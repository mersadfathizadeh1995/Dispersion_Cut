================================================================================
  1. PROJECT OVERVIEW
================================================================================

DC Cut is a seismic dispersion curve analysis app (PySide6 + matplotlib).
Report Studio (packages/report_studio/) is the plotting/figure sub-app
with a composable multi-sheet canvas, hierarchical layer tree, and
GeoFigure-style project persistence.

The user wants a Photoshop/AutoCAD-like GUI: center canvas with sheets,
side panels for data tree (left) and settings (right), top menu bar.

================================================================================
  3. KEY DATA MODELS (core/models.py)
================================================================================

OffsetCurve:
  - uid (auto-generated via _short_uid() in __post_init__ if empty)
  - name, frequency, velocity, wavelength (numpy arrays)
  - Style: color, line_width, marker_size, line_style, marker_style, etc.
  - spectrum_uid -> links to SpectrumData
  - subplot_key -> which subplot it belongs to
  - point_mask -> boolean mask for toggling individual points

AggregatedCurve:
  - uid, name
  - bin_centers, avg_vals, std_vals (numpy arrays — recomputed, not saved)
  - shadow_curve_uids: List[str] — UIDs of component OffsetCurves
  - Style: avg_color, avg_line_width, uncertainty_mode, uncertainty_alpha, etc.
  - Visibility: avg_visible, uncertainty_visible, shadow_visible, shadow_alpha
  - Config: num_bins, log_bias, x_domain ("frequency" | "wavelength")
  - subplot_key

SubplotState:
  - key, name, stype
  - curve_uids: List[str], aggregated_uid: str
  - Axis: x_domain, x_range, y_range, auto_x, auto_y, x_scale, y_scale
  - Typography: font_family, title_font_size, axis_label_font_size, tick_label_font_size
  - Tick format: x_tick_format, y_tick_format
  - Labels: x_label, y_label
  - Legend: legend_visible, legend_position, legend_font_size, legend_frame_on

SheetState:
  - name, curves: Dict[uid->OffsetCurve], spectra: Dict[uid->SpectrumData]
  - aggregated: Dict[uid->AggregatedCurve]
  - subplots: Dict[key->SubplotState]
  - Grid: grid_rows, grid_cols, col_ratios, row_ratios, hspace, wspace
  - Figure: figure_width, figure_height, canvas_dpi
  - Paths: pkl_path, npz_path
  - legend: LegendConfig, typography: TypographyConfig
  - Methods: add_curve(), remove_curve(), move_curve(), add_aggregated(),
    remove_aggregated(), move_aggregated()

================================================================================
  4. KEY PATTERNS & CRITICAL KNOWLEDGE
================================================================================

UID PERSISTENCE (v2.8.3 fix — CRITICAL):
  - OffsetCurve.__post_init__ generates new UID if uid is empty
  - When PKL data is reloaded, fresh OffsetCurve objects get NEW UIDs
  - project_v4._apply_curve_settings() MUST restore saved UID so that
    subplot.curve_uids and aggregated.shadow_curve_uids references match
  - Without this, aggregated figures disappear after save/load

SIGNAL FORWARDING (data_tree.py):
  - _DragTreeWidget (inner) has its own signals: curve_moved, aggregated_moved
  - DataTreePanel (outer) has matching signals
  - Inner signals MUST be forwarded: _tree.signal.connect(panel.signal.emit)
  - Both must declare the signal — forgetting on inner widget causes silent fail

SUBPLOT x_domain SYNC:
  - SubplotState.x_domain controls ALL rendering (axis labels, shadow curves)
  - AggregatedCurve.x_domain controls which binning function is used
  - MUST stay in sync: set sp.x_domain = agg.x_domain when adding or changing

PLUGIN SYSTEM:
  - Plugins auto-register on import (module-level code in each plugin file)
  - Registry: core/figure_types.py -> FigureTypeRegistry
  - Each plugin has: type_id, display_name, load_data(pkl_path) method
  - load_data returns {"curves": [...], "aggregated": AggregatedCurve|None}

PLOT REGISTRATION CHECKLIST (when adding new figure types):
  1. Plugin in core/plugins/ (inherit FigureTypePlugin, implement load_data)
  2. Register in figure_types registry (auto via module-level code)
  3. Serialization in project_v4.py (save/load must handle new curve types)
  4. Tree display in data_tree.py (item types, checkbox propagation, context menu)
  5. Drag-drop support (both inner widget + panel signals)
  6. Settings panel (if new settings are needed)
  7. Tests covering all the above

PROJECT PERSISTENCE (project_v4.py):
  - Directory-based: project_dir/project.json + sheets/*.json
  - Saves settings ONLY — no numpy arrays (prevents 500MB+ files)
  - On load: skeleton from JSON + reload arrays from PKL/NPZ + match by name
  - Flow: save_sheet() -> load_sheet_skeleton() -> reload_and_apply()
          -> _restore_aggregated()
  - Recent projects tracked in QSettings

RENDERING PIPELINE:
  - SheetRenderer.render_sheet(sheet, fig) -> clears fig -> LayoutBuilder ->
    for each subplot: CurveDrawer + AggregatedDrawer + SpectrumDrawer
  - renderer.py uses sp.x_domain to decide x-axis data and labels
  - aggregated_drawer.py handles avg line, uncertainty band/sticks, shadow curves

DATA TREE HIERARCHY:
  Subplot 1 (_TYPE_SUBPLOT)
    +-- Average (_TYPE_AGG_GROUP, draggable)
    |   +-- Average Line (_TYPE_AGG_AVG, checkable)
    |   +-- Uncertainty (_TYPE_AGG_UNC, checkable)
    |   +-- Shadow Curves (_TYPE_AGG_SHADOW_GROUP)
    |       +-- Offset +5m (_TYPE_AGG_SHADOW, checkable)
    |       +-- Offset +10m (_TYPE_AGG_SHADOW, checkable)
    +-- Offset +15m (_TYPE_CURVE, draggable, checkable)
    +-- Offset +20m (_TYPE_CURVE, draggable, checkable)

  _find_subplot_ancestor() walks up tree to find subplot root for drag targets

NUMPY SAFETY:
  - Never use `np_array or fallback` — truth value is ambiguous
  - Use helpers like `_ndarray_or_empty(result)` that check `if result is None`

QT COMPATIBILITY:
  - DC Cut uses matplotlib.backends.qt_compat (not PySide6 directly)
  - Studio has its own qt_compat.py for enum wrappers across PyQt5/6/PySide2/6

TESTING:
  - pytest-qt with matplotlib FigureCanvasQTAgg
  - Call qapp.processEvents() at end of test to flush draw_idle timers
  - matplotlib Agg backend for headless testing
